🔗 Auto-Restore Link — dedicated registry script

This solves "I refresh and everything's gone." A separate, dedicated Apps Script — completely independent from the one that saves meter entries — keeps a small registry of your projects (just their names + Sheet IDs). Keeping it separate means redeploying your entries script for other changes can never break this, and vice versa. Set it up once; opening the app via the special link this generates automatically pulls your project list back in.

1. Create one new, blank Google Sheet — just for this registry, don't reuse a project Sheet (you can switch to a different one later just by changing the Sheet ID below — nothing hardcoded).
2. Copy its ID from that Sheet's own URL.
3. Go to script.google.com → New project — this must be a different, separate Apps Script project from your entries one.
4. Paste the Registry Script code below exactly as-is — nothing to fill in inside the code itself.
5. Deploy → New deployment → Web app → Execute as: Me → Access: Anyone → Deploy. Run doPost once manually first if this is a fresh script, to grant Sheets access.
6. Paste that URL and your Registry Sheet ID below, along with your entries script URL (same one used on project cards), then tap Save & Generate My Link and bookmark it.

New: Restricted Access (opt-in per project). Turn it on for a project and its card gets a "Grant Access" form — pick a device, set a duration, and only that device can join it, for that long. Manage it by editing the Access - [Project Name] tab this creates inside your Registry Sheet — the Status dropdown (Allowed / Restricted / Paused) takes effect the next time that device opens the app or reopens the project. Projects that never opt in keep working exactly as before, no change.

☁ Cloud Sync — Meter Entries Apps Script

You only do this once — total, not per project. This Web App handles every project's Sheet for meter records, dispatcher lists, and photos — the Sheet ID is sent along with each request instead of being fixed inside the script. This is a separate script from the Auto-Restore registry above — keeping them apart means redeploying this one for a records/photo change can never break your saved project list, and vice versa.

1. Go to script.google.com → New project (a different project from the registry one above).
2. Delete any starter code, paste the script below exactly as-is — nothing to fill in or replace.
3. Deploy → New deployment → Type: Web app → Execute as: Me → Who has access: Anyone.
4. Copy the URL (ends in /exec).
5. For every project — new or existing — paste that same URL into the project's card, along with that project's own Sheet ID. That's it, nothing else to set up per project.

One requirement: every Sheet ID you use must belong to (or be shared with) the same Google account that deployed this script. Photos go into one master Drive folder called Meter Receiving Photos, with a subfolder per project named ProjectName - Date - MRV No - Device Name. Filenames include the meter number and the device that took the photo. If you already have sheets from before this version: add these header cells to the end of that Sheet's DMIS Receiving row 1 — MR/MRV No., Reading Photo URLs, Condition Photo URLs, OGRA Photo URLs, MRC Photo URLs — new Sheets get this automatically. New: every save now also writes a condensed, anomaly-only row into a second tab called List For Print in the same Sheet, now with a Remarks column for your free-text notes — DMIS Receiving keeps every field exactly as before, unchanged. This only applies to entries saved from now on; nothing already in your Sheets gets backfilled. Page numbers and a repeating print header/footer can't be set by this script — Google's API simply doesn't expose those settings, they only exist in the Print dialog. One-time manual fix per Sheet: open the List For Print tab → File → Print → under "Headers & footers," turn on page numbers and set repeated top/bottom text if you want it → this choice is remembered by the Sheet going forward, no need to redo it per entry. New: this same script now also handles Dispatch/Scrap/Irrepairable list entries from the "📦 Dispatch/Scrap Lists" tab — they write to a separate Dispatch-Scrap List tab in whichever Sheet that list is pointed at, with their own photo folder naming (listprep_meterNo_device_...). Nothing about Receiving Inspection changes. Note: unlike Receiving, there's no "Sync from Sheet" read-back for these yet — each device only sees what it has saved locally so far. Each List Prep project also gets its own List For Print-[Project Name] tab — a 4-column-block printable grid (Sr. No. / Meter Type / Meter No., 50 rows per block, 200 per page), Meter Type shown as Alias-Category, matching the official register format. To grant Drive access: in the function dropdown next to ▶ Run, select authorizeDriveAccess — not doPost, since doPost exits before touching Drive when run manually and will never trigger the permission prompt — then click Run and approve the popup.
Known limitation: writes (Save Entry, dispatcher upload, photo uploads) go through doPost, and the browser still can't read that response — it marks "Sent (unconfirmed)". Reads (Join via Code, Sync from Sheet) go through doGet via a JSONP trick, which does work — so photo links only appear in the app itself (not just the Sheet) after you tap Sync from Sheet on that project. Keep exporting Excel per project as backup regardless. OneDrive auto-export has no equivalent no-login trick — that genuinely needs the Phase-2 backend with Microsoft sign-in.
